Goal & Value


The goal of this application is to provide a shopping interface for Brown Dining Services (BDS).

Currently, BDS has a website that lists the dining hall options. However, this operates solely as a menu. My website aims to provide a way for students to view and order their food, which may not require as much in-person interaction.

This is also a good tool for meal planning, as typically there is only a short description given for each item. This website allows students to get a better idea of offerings that are available and plan out what they would like to eat. This is a general marketplace app, not taking into consideration all-you-can-eat or point systems. It provides an estimation of the real-world prices of Brown's offerings.

For students who are not on campus (like myself), this is a fun interface to provide them with some familiarity. Students may be missing Brown's dining options, so this is an interactive way to reminisce upon better times.


Design


Components:

  • Location Filter
    1. Andrews
    2. Ivy Room
    3. Jo's (or Jos?)
    4. Ratty
    5. VDub
  • Meal Filter
    1. Breakfast
    2. Lunch
    3. Dinner
  • Sort
    1. Random
    2. Price Low to High
    3. Price High to Low
  • Combineable filters and sorting
  • Item Cards
    1. Item Illustration
    2. Location
    3. Meal
    4. Price
    5. Add to Cart Button (can be clicked multiple times)
  • Cart Aggregator
    1. Displays card for each added item
    2. Remove (if > 1) and add items
    3. Remove item completely from cart
    4. Sums total price- updated with any interaction
    5. Clear entire cart

Usability Principles

  • Logical page setup
  • Page is responsive
  • Indicators for what settings are selected
  • Text indicates if no items fit parameters or if cart is empty
  • Buttons respond to hover
  • Colors chosen for adequate contrast
  • Distinct item cards
  • Padding and margins to keep content within high visibility region
