Brown Dining Services




The goal of this project, "Development", was to use React to create a list website that would aggregate some value for the list items. I decided to create a shopping interface for Brown Dining Services (BDS).

I was inspired by Brown's very specific, sometimes underwhelming dining options. Despite having been away from campus for so long, these foods remain very clear in my mind- during quarantine, my friends and I had a very spirited argument over a Brown Dining Options Tier List (my personal ranking is below). It seems that people are really passionate about food at Brown.



Currently, BDS has a website that lists the dining hall options. However, this operates solely as a menu. My website aims to provide a way for students to view and order their food, which may not require as much in-person interaction.

This is also a good tool for meal planning, as typically there is only a short description given for each item. This website allows students to get a better idea of offerings that are available and plan out what they would like to eat. This is a general marketplace app, not taking into consideration all-you-can-eat or point systems. It provides an estimation of the real-world prices of Brown's offerings.

For students who are not on campus (like myself), this is a fun interface to provide them with some familiarity. Students may be missing Brown's dining options, so this is an interactive way to reminisce upon better times.


Design


Components


The project required filter and sorting functions. I implemented a Location filter (Andrew, Ivy Room, Jo's, Ratty, VDub) and Meal filter (Breakfast, Lunch, Dinner). I also enabled sorting by ascending and descending price.



I created an Item Card for each item. Each card displays an illustration, location, meal, and price, followed by an Add to Cart button.

The last component is the aggregator- in this case, a cart. The cart contains each added item, with the option to add/remove a unit, as well as remove an item entirely. There is also an option to clear the entire cart. The total sum is displayed at the bottom.

Usability Principles


  • Logical page setup
  • Page is responsive
  • Indicators for what settings are selected
  • Text indicates if no items fit parameters or if cart is empty
  • Buttons respond to hover
  • Colors chosen for adequate contrast
  • Distinct item cards
  • Padding and margins to keep content within high visibility region



Item Illustrations

I used Procreate to create illustrations for each item available for purchase. These are relatively minimalistic but I tried to include key details, like the snap peas in the Ratty miso bowl and the unchopped Jo's salad.

Conclusion


After getting used to React, making this website was really enjoyable. There are so many possibilities, which makes it a really useful tool. The HTML for this project only contained one element, so I think experimenting more with using HTML and React would be helpful in the future.
I also had so much fun designing the website- it wouldn't really be feasible in reality, but I do think it has some promising ideas. It's definitely something that doesn't exist at this moment. I enjoyed "ordering food", and am excited to learn more of what's possible with React.
